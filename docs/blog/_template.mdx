Un párrafo corto de apertura (lead). No pongas un `# Título` aquí — el título
sale de la frontmatter. Arranca directo con una idea que invite a seguir
leyendo.

<SectionTitle id="intro">Introducción</SectionTitle>

Contextualiza el post. ¿Qué asumes del lector? ¿Por qué ahora? Mantén esta
sección breve — 2–4 párrafos máximo.

<Callout type="info">
  Usa **callouts** para marcar aclaraciones o pro-tips que no caben en el flujo
  principal. Quita este ejemplo cuando no lo necesites.
</Callout>

<SectionTitle id="problema">El problema</SectionTitle>

Describe el problema concreto que vas a resolver. Muestra código, pantallas o
comandos reales — evita abstracciones hasta que el lector sepa _por qué_
importan.

```bash
# Las fences con lenguaje se renderizan como <CodeBlock> automáticamente.
pnpm install
pnpm dev
```

<SectionTitle id="solucion">La solución</SectionTitle>

Aquí va el cuerpo técnico. Algunos ejemplos de patrones útiles — borra los
que no necesites:

  sub-tema merece su propia entrada en el TOC, usa `<SectionTitle>` con un `id`
  en la frontmatter en vez de `<SubSection>`.
</SubSection>
